**SurrealDB 3.0 syntax discoveries:**
- `FLEXIBLE TYPE object` → `TYPE object FLEXIBLE` (keyword order changed in v3.0)
- `SEARCH ANALYZER` → `FULLTEXT ANALYZER` (renamed in v3.0)
- Full-text indexes are per-field only (cannot index multiple fields in one DEFINE INDEX)
- SCHEMAFULL tables in v3.0 reject extra fields with errors (no silent filtering)

**Key schema design decisions applied:**
- `companyId` dropped from all company DB tables (implicit via database isolation)
- Cross-DB references stored as `TYPE string` (SurrealDB cannot do cross-namespace record links)
- `gmapCoord` + `geoHash` → native `TYPE geometry<point>` (SurrealDB handles geo natively)
- Embedded objects for 1:1 data (booking_policy, opening_schedule, reservation_config)
- Separate `date_override` table for staff (scalability over time)
- Norwegian snowball stemmer for full-text search on discovery listings
- `deleted_at` soft delete on all major tables (GDPR audit trail)
- `audit_log` table added to platform DB (GDPR compliance tracking)
- `booking_ref` table added to users DB (cross-DB booking feed for consumers)
