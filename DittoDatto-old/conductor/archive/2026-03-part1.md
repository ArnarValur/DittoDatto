**What was accomplished:**

2. ✅ **Opening schedule wiring** — `[slug].vue` wasn't passing `:opening-schedule` or `:reservation-config` to `BookingSlideover`. Fixed.

4. ✅ **Double-booking collision** — Server-side check added to `createHold()`. Code done, but **6x overbooking still reproduced** — see critical blocker.
5. ✅ **Shared time utilities** — Extracted `getStoreNow()` + `isSlotInPast()` to `core/time.ts`.
6. 🔴 **DISCOVERED: Reservation flow wiring gap** — Frontend calls `/appointments/holds` for ALL bookings including restaurants. The `/reservations` endpoint (with capacity checks, table assignment) exists but is never called.

**Key learnings:**
- **Server tests passed but tested wrong layer** — 25 tests all green, but the actual bug was in the Vue component, not the engine.
- **"Are we online?" was verified at server level, not end-to-end** — The engine responds correctly, but the frontend doesn't call the right endpoints for reservations.
- **Stop patching symptoms** — Multiple individual bug fixes were applied before discovering they all share one root cause: incomplete frontend wiring.
