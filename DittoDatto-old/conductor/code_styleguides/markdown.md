# Markdown Style Guide

## Principles
*   **Clarity:** Documentation should be easy to read and follow.
*   **Consistency:** Use the same formatting for headers, lists, and code blocks.
*   **Visuals:** Use diagrams (Mermaid) and images where they add value.

## Formatting
*   **Headers:** Use `#` for the main title, `##` for sections, and `###` for sub-sections.
*   **Lists:** Use `-` for unordered lists and `1.` for ordered lists.
*   **Code Blocks:** Always specify the language for syntax highlighting (e.g., ```typescript).
*   **Emphasis:** Use **bold** for important terms and *italics* for emphasis.

## Organization
*   **TOC:** Include a Table of Contents for long documents.
*   **Links:** Use descriptive text for links, not just "click here".
*   **Callouts:** Use blockquotes or specific Markdown extensions (if supported, e.g., `> [!NOTE]`) for highlights.
