The Noona HQ API is the comprehensive business management API for service providers. It provides full control over all aspects of running a service business, from managing staff and customers to handling bookings, payments, and reporting.

## Key Features

The HQ API provides access to:

- **Business Management** - Manage companies, locations, and enterprise settings
- **Staff & Resources** - Handle employees, roles, and resource scheduling
- **Customer Management** - Complete CRM functionality for customer relationships
- **Booking & Scheduling** - Full control over events, time slots, and blocked times
- **Financial Operations** - Payments, transactions, sales, and reporting
- **Product & Service Catalog** - Manage services, products, and pricing
- **Integrations** - OAuth applications, webhooks, and third-party integrations
- **Advanced Features** - Metrics, goals, custom properties, and automation rules

Browse the sidebar to explore all available endpoints, organized by resource type:

- **Companies** - Create and manage business locations
- **Employees** - Staff management and scheduling
- **Customers** - CRM and customer data management
- **Events** - Booking and appointment management
- **Event Types** - Service and product catalog
- **Payments & Transactions** - Financial operations
- **Webhooks** - Real-time event notifications
- **OAuth Applications** - Build and manage integrations
- **Metrics & Reports** - Business analytics and insights
- And 40+ more resource categories...

## Getting Started

The HQ API requires [OAuth 2.0 authentication](https://docs.noona.is/docs/authentication/oauth) with appropriate scopes based on the operations you need to perform. Make sure to request the necessary permissions when setting up your application.

For building integrations that other businesses can install, check out our [App Store guide](https://docs.noona.is/docs/app-store).
