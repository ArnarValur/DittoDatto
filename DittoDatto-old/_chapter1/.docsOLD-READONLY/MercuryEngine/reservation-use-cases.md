# Reservation System — Use Cases

## 👤 Customer (Public Marketplace)

### Browsing & Booking
1. User can open stores and restaurants to browse and explore services.
2. User can click Book button and select a Service to book.
3. User can Book many Services together if "Allow Multi-Select" is enabled on the Service-Group.
4. User can click on Time-Slot.
5. User can leave note with the Bookings.
6. User can see the Booking on their User Page.
7. User can cancel booking (Companies Cancel Policies Apply)
8. User can open Booking and click "Im 5min late" button.
9. User can add establishment in Favourites.
### Managing Reservations
1. User can click Book on Restaurant or Venue store-pages and select Service (that has Reservation as Booking-Type) to make a reservation.
2. User can click on the Service-Cards on the front store-page and it'll open that booking flow preselected.
3. User can book more than one duration "one unit?" of a service with table reservation, as in: I should be able to book one or two hours.
4. User can click Waiting-List if the Time-Slot is not available.
5. User can cancel reservation (Companies Cancel Policies Apply).
6. User can notify being late button.
